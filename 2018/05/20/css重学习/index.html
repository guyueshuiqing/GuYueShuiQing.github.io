HTML面试

1、doctype的意义是什么

这是有历史背景的，很久以前ie有一些自己的渲染模式。最典型的就是盒子模型，我么都知道一个元素他如果是width宽度的话那么他实际占的宽度还会包括他的padding和margin ，ie来说如果设置的宽度为200px，padding 10px，那么整个区域可能是180,但是标准的盒子模型它事实上宽度可能是220，后来就出现了一个解决方案，如果带上doctype那么ie就会以标准模式渲染。

所以他的意义：

  • 让浏览器标准模式渲染（不只是IE）
  • 让浏览器知道元素的合法性(写的是html4那么有的元素可能不合法，如果是Xhtml那么元素不闭合，属性值不加引号等就是不合法的)

2、HTML、XHTML、HTML5的关系

  • HTML是属于SGML的一个应用（SGML是一个非常通用的标记语言）
  • XHTML是属于Xml，是HTML进行HTML严格化的结果
  • HTML5不属于SGML和XML，比XHTML宽松

3、HTML5有什么变化

  • 新的语义化的元素（section、article、header、footer、nav等，能方便的写出更加有语义化的元素）
  • 表单增强
  • 新的API（离线、音视频、图形、实时通信、本地储存、设备能力）
  • 分类和嵌套规则的变更（a中可以嵌套块级元素）

4、em和i有什么区别

  • em是语义化标签，表示强调

  • i是一个纯样式的标签，表示斜体

  • html中i不推荐使用，一般用作图标

5、语义化的意义是什么

  • 开发者容易理解，易于维护
  • 机器容易理解（搜索、盲人读屏软件）
  • 有益于SEO
  • semantic microdata（进行标记）

6、哪些元素可以自闭和

  • 表单元素input
  • 图片img
  • br、hr
  • meta、link

7、HTML和DOM的关系

  • HTML是死的（所谓的结构只是看起来有结构而已）
  • DOM是有HTML解析而来，是活的，是浏览器内存中的一个结构，是一个树形结构
  • js真正去维护的是DOM

8、property和attribute的区别

  • attribute是死的
  • property是活的

9、form的作用

  • 直接提交表单
  • 使用submit和reset按钮
  • 便于浏览器保存表单
  • 第三方库可以整体提取值
  • 第三方库进行表单验证

CSS布局（重点）

盒子模型：

  • content+padding+border+margin

  • 默认情况：高度、宽度只对内容生效

  • 占用空间：content+padding+border

position：

  • static： static定位是HTML元素的默认值，即没有定位，元素出现在正常的流中。因此这种定位就不会受到top、bottom、left、right的影响。
  • fixed： fixed定位是指元素的位置相对于浏览器窗口是固定位置，即使窗口是滚动的他也不会滚动，而且fixed定位使元素的位置与文档流无关，因此不占据空间，所以会和其他元素发生重叠。
  • relative： 相对定位元素的定位是相对他自己的正常位置定位。
  • absolution：绝对定位的元素相对于最近的已定位父元素，如果元素没有已定位的父元素，那么它的位置相对于。

float：

  • 元素浮动

  • 脱离文档流

  • 但不脱离文本流（占据文字位置）

  • 对自身影响：

    • 形成块（BFC）
    • 位置尽量靠上

  • 对兄弟的影响

    • 上面一般贴非float元素
    • 旁边一般贴float元素
    • 不影响其他块级元素的位置
    • 影响其他块级元素内部的文本

  • 对父级元素的影响

    • 从父布局消失
    • 高度塌陷

  • 清除浮动

    • 经典清除浮动

      .clearfix:after{
    content: '';
    display: block;
    height: 0;
    clear: both;
    visibility: hidden;
}
/* 兼容IE6 IE7  zoom：1； 触发haslayout属性 */
.clearfix{
    zoom: 1;
}

    • 改写：

      .clearfix:after{
    content: '';
    display: table;
    clear: both;
}

    • 解决margin塌陷

      .clear-fix:before,
.clear-fix:after{
content: '';
  display: table;
}

.clear-fix:after{
  clear: both;
}

  • 盒子是并列的

CSS布局面试题

1、实现两栏或者三栏的布局方法

  • 表格布局
  • float+margin布局
  • inline-block布局
  • flexbox布局

2、position：absolute|fixed有什么区别

  • 前者相对最近的absolute、relative定位
  • 后者相对屏幕（viewport）

3、display：inline-block怎么会有间隙、怎么处理

  • 原因：字符间距
  • 解决
    • 消灭字符（设置父级font-size为0）
    • 消灭间距（HTML中注释换行代替换行，删掉换行符）

4、清除浮动

  • 原因：浮动的元素不会占据父元素的布局空间，可能浮动元素会超出父元素对其他元素造成影响

  • 解决：

    • 让盒子负责自己的布局（触发BFC）
      • overflow：hidden|auto
      • ::after{clear:both}

5、如何适配移动端页面

  • 把页面viewprot适配
  • rem/viewport/media query
  • 设计上：隐藏、折行、自适应

CSS动画

transition补间动画

  • 位置-平移（left | right | margin | transform）
  • 方位-旋转（transform）
  • 大小-缩放（transform）
  • 透明度（opacity）
  • 其他-线性变换（transform）
.container{
    width: 100px;
    height: 100px;
    background: red;
    /* transition: all 1s; */
    transition: width 1s, background: 3s;
    /* transition-timing-function: linear  */
}
.container: hover{
    width: 800px;
    backgroud: green;
}

keyframe关键帧动画

  • 相当于多个补间动画
  • 与元素的状态无关
  • 定义更加灵活
.container{
    width: 100px;
    height: 100px;
    background: red;
    animation: run 1s;
    /* 设置循环次数 无线循环 */
    animation-iteration-count: infinite;
}

@keyframes run{
    0%{
        width: 100px;
    }
    50%{
        width: 600px;
    }
    100%{
        width: 100px;
    }
}

逐帧动画

  • 适用于无法补间的动画
  • 资源较大
  • 使用steps（1）
.container{
    width: 100px;
    height: 100px;    
    background: url(./animal.png) no-repeat;
    animation: run 1s infinite;
    /* 让动画不补间，就每个帧就一个画面 */
    transition-timing-function: steps(1);
}
/* 设置雪碧图的位置不断切换图片达到动画的目的 */
@keyframes run{
    0%{
        backgrund-position: 0 0;
    }
    12.5%{
        backgrund-position: -100px 0;
    }
    25%{
        backgrund-position: -200px 0;
    }
    37.55%{
        backgrund-position: -300px 0;
    }
    50%{
        backgrund-position: 0 -100px;
    }
    62.5%{
        backgrund-position: -100px -100px;
    }
    75%{
        backgrund-position: -200px -100px;

CSS预处理器

什么是CSS预处理器

  • 基于css的一种语言
  • 通过工具编译成css
  • 添加许多css不具备的特性
  • 提升css文件的组织

CSS预处理器的功能

  • 嵌套 反映层级和约束
  • 变量和计算 减少代码重复
  • Extend和Mixin代码片段
  • 循环 适用于复杂有规律的样式
  • import css文件模块化

CSS预处理器框架

  • Sass-Compass
  • Less-Lesshat|EST
  • 提供现成的mixin
  • 类似js封装库封装各种功能

  • 提高代码的复用性
  • 提升可维护性
